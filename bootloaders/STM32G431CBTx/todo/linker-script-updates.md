# Modify Linker Script

We need to modify the linker script which is generated by STM32CubeIDE to properly set the requirements for our flash memory usage for the bootloader.
Primary need here is to properly route memory region defintions to the application via `flash_base_addr__` and `application_start_addr__`, 

I confirmed that the startup assembly defines the vector table contents but does not depend on an absolute address in flash memory. Setting the VTOR pointer is sufficient for properly mapping interrupts in a target with modified ORIGIN.
